一、产品概述

产品定位

为企业级后台管理系统提供基于RBAC模型的权限管控平台，实现用户、角色、权限、组织架构的精细化管理。

目标用户

系统管理员、部门负责人、IT运维人员、安全合规专员，以及需要精细化权限管控的企业客户。

核心价值

降低80%权限管理成本，实现权限配置可视化、数据范围精细化、操作审计合规化。

区域说明

① 面包屑导航

显示当前页面路径，支持点击返回上级菜单。格式：系统管理 / 用户管理

② 搜索筛选区

④ 数据列表

展示用户信息表格，支持：排序、分页、行内操作（编辑/禁用/重置密码）

功能详细说明

1. 用户列表展示
P0 - 核心
展示字段
  • • 用户头像 + 姓名（头像取姓名首字）
  • • 登录账号
  • • 所属部门（显示完整部门路径）
  • • 角色标签（多个角色以标签展示）
  • • 状态（启用/禁用/锁定）
  • • 最近登录时间
交互规则
  • • 默认按创建时间倒序排列
  • • 支持点击表头排序
  • • 每页默认20条，支持切换10/50/100条
  • • 行 hover 显示操作按钮
  • • 长按表头可拖拽调整列宽
2. 搜索与筛选
P0 - 核心
搜索条件
关键词搜索

支持：姓名、账号、手机号、邮箱模糊搜索

部门筛选

树形下拉选择，支持多选

状态筛选

全部/启用/禁用/锁定

搜索逻辑：关键词实时搜索（防抖300ms），筛选条件点击"查询"后生效。支持组合筛选条件，筛选结果可导出。
3. 新建用户
P0 - 核心
弹窗表单字段
姓名 *必填，2-20字符
登录账号 *必填，字母开头，4-20位
初始密码 *必填，默认生成可修改
手机号选填，需校验格式
邮箱选填，需校验格式
所属部门 *必填，树形选择器
角色 *必填，可多选
状态默认启用
业务规则
  • • 账号唯一性校验，已存在则提示
  • • 初始密码默认8位随机字符
  • • 创建成功后发送通知（站内信+邮件）
  • • 首次登录强制修改密码
  • • 记录创建人、创建时间
校验规则 成功提示 错误提示 确认弹窗
4. 批量操作
P1 - 重要
批量启用

将选中禁用的用户改为启用状态，需二次确认

批量禁用

将选中启用的用户改为禁用状态，需二次确认，已登录用户强制下线

批量删除

软删除选中用户，需二次确认，删除后无法登录

限制规则：每次最多操作100条；操作超级管理员需要额外确认；操作结果以Toast提示，失败项显示详情。

区域说明

① 角色列表

展示所有角色卡片，包含：角色名称、描述、用户数。点击切换右侧配置面板。

② 角色信息

显示当前选中角色的名称和描述，支持编辑。提供克隆、删除、保存操作按钮。

③ 权限树

树形结构展示所有权限，支持：勾选父级自动全选子级、单独勾选子级、操作类型细粒度控制。

④ 数据范围

设置该角色用户可查看的数据范围：全部数据/本部门及子部门/仅本部门/仅个人。

功能详细说明

1. 角色列表
P0 - 核心
角色卡片信息
  • • 角色图标 + 名称
  • • 角色描述（1行截断）
  • • 关联用户数
  • • 系统角色标识（不可删除）
交互规则
  • • 点击卡片切换右侧配置
  • • 当前选中卡片高亮显示
  • • hover显示编辑/删除按钮
  • • 系统角色隐藏删除按钮
2. 权限配置树
P0 - 核心

四级权限体系：

  • • 菜单权限 - 控制左侧导航菜单的显示/隐藏
  • • 按钮权限 - 控制页面内操作按钮（增删改查导）
  • • 数据权限 - 通过数据范围设置控制
  • • 接口权限 - 后端控制，前端无需配置
勾选逻辑：勾选父节点自动勾选所有子节点；取消父节点自动取消所有子节点；部分勾选子节点时父节点显示半选状态。
3. 数据范围配置
P1 - 重要
全部数据
可查看系统中所有数据，不受部门限制。仅超级管理员可用。
本部门及子部门
可查看本部门及所有下级部门的数据。
仅本部门
仅可查看本部门数据，不包括子部门。
仅个人数据
仅可查看自己创建或负责的数据。

用户状态流转

⇄

已禁用
管理员禁用

状态流转规则：创建后首次登录自动激活；密码错误5次锁定30分钟；管理员可手动禁用/启用；禁用用户无法登录

权限校验流程

1
用户发起请求

用户访问页面或点击按钮，前端发起API请求，携带JWT Token

2
身份认证

后端验证Token有效性，获取用户ID和角色信息

3
权限校验

检查用户角色是否拥有该API权限，校验数据范围是否符合

4
结果返回

有权限：返回数据；无权限：返回403，记录审计日志

四、核心业务规则

用户管理规则

  • 账号全局唯一，不可重复
  • 一个用户可拥有多个角色
  • 用户必须属于一个部门
  • 禁用用户立即踢出所有登录
  • 删除为软删除，数据保留

角色权限规则

  • 超级管理员拥有所有权限
  • 系统角色不可删除
  • 角色权限修改实时生效
  • 无角色用户无法登录
  • 权限取角色并集

组织架构规则

  • 支持无限层级部门
  • 部门编码全局唯一
  • 有子部门不可删除
  • 支持部门合并与拆分
  • 部门变更影响数据范围

数据权限规则

  • 数据范围与角色绑定
  • 多角色取最大范围
  • 跨部门数据需单独授权
  • 数据权限后端强制执行
  • 操作审计记录全量数据

五、交互与反馈

操作反馈规范

✓
成功提示
操作成功："创建用户成功" Toast提示，2秒自动消失
✗
错误提示
操作失败：表单字段标红，显示具体错误信息
!
确认弹窗
危险操作：删除前弹窗确认，显示影响范围
i
加载状态
数据加载：显示Skeleton骨架屏或Loading动画

异常处理规范

网络异常

请求超时或网络断开时，显示"网络异常，请检查网络连接"，提供重试按钮

权限不足

访问无权限页面时，显示403页面，提示"您没有权限访问此页面"

数据不存在

访问已删除数据时，显示404页面，提示"数据不存在或已被删除"

服务端错误

服务器500错误时，显示"系统繁忙，请稍后再试"，记录错误日志
