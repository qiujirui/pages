设计目标

  • 实现细粒度权限控制，支持菜单/按钮/数据/API四级权限
  • 支持多租户组织架构，实现数据范围隔离
  • 提供完整的审计追踪能力，满足合规要求
  • 支持动态权限配置，无需重启服务

02 项目背景

现有痛点

  • • 权限控制粒度粗，只能控制到菜单级别
  • • 用户与权限直接绑定，维护成本高
  • • 缺乏数据权限控制，无法隔离敏感数据
  • • 权限变更需重启服务，影响业务连续性
  • • 无操作审计日志，无法满足合规要求

预期收益

  • • 细粒度权限控制，提升系统安全性
  • • 角色批量授权，降低管理成本80%
  • • 数据范围隔离，保护核心业务数据
  • • 动态权限配置，实时生效无需重启
  • • 完整审计追踪，满足等保合规要求

RBAC 核心优势

🔐

按需分配，防止越权
🔄
职责分离
敏感操作需多人授权
📊

多租户数据范围控制
📝

操作留痕可追溯

07 安全设计

认证安全

  • JWT Token认证，支持刷新Token机制
  • 密码采用BCrypt加密存储
  • 登录失败次数限制（5次锁定30分钟）
  • 强制密码复杂度（8位以上+大小写+数字）
  • 定期密码更换提醒（90天）

授权安全

  • 接口级别权限校验（后端二次校验）
  • 数据范围动态过滤（SQL拦截注入）
  • 敏感操作二次确认（删除、批量操作）
  • 越权访问自动拦截并告警
  • Session超时自动登出（30分钟无操作）

传输安全

  • 全站HTTPS加密传输
  • API请求签名验证防篡改
  • 请求频率限制（防暴力破解）
  • CORS跨域白名单控制

数据安全

  • 敏感数据脱敏展示（手机号、身份证）
  • 数据库敏感字段加密存储
  • 数据库访问权限最小化
  • 定期数据备份与恢复演练
