工作模式详解

Craft / Plan / Ask 三种模式的使用场景与切换策略

⚡
Craft 模式：你说，我做

适用场景：任务明确、需求清晰、需要快速交付结果的情况

# 示例对话 用户：把昨天的门店销售数据按区域汇总，生成Excel发给我 WorkBuddy：立即执行 → 连接数据库 → 查询数据 → 生成透视表 → 输出Excel

最佳实践：

  • 提供明确的文件名、路径、格式要求
  • 复杂任务分步骤描述，避免一次性提过多要求
  • 涉及敏感操作（删除、修改）时主动确认
📝
Plan 模式：先规划，后执行

适用场景：复杂项目、跨系统对接、需要多步骤协作的情况

# 示例对话 用户：我想搭建一个门店业绩预警系统 WorkBuddy：先分析需求 → 设计方案 → 确认技术栈 → 分阶段实施 → 第1阶段：数据源对接 → 第2阶段：预警规则配置 → 第3阶段：通知机制实现 → 第4阶段：测试上线

最佳实践：

  • 先让AI给出完整方案，确认后再执行
  • 对关键技术决策保留否决权
  • 每完成一个里程碑检查一次成果
💬
Ask 模式：只咨询，不动手

适用场景：学习新知识、评估方案可行性、获取专业建议

# 示例对话 用户：我想了解餐饮行业常用的会员RFM模型怎么落地 WorkBuddy：仅提供知识解答，不生成任何文件或代码 → 解释RFM模型概念 → 给出餐饮场景的计算方法 → 提供SQL实现示例 → 建议后续实施步骤

最佳实践：

  • 先问清楚再动手，避免返工
  • 涉及合规、财务等敏感问题时优先咨询
  • 新技术选型时多收集信息

场景一：门店数据分析

从数据获取到洞察输出的完整工作流

典型工作流
1

数据获取

从POS系统导出销售数据，或通过API直接拉取数据库数据。支持CSV、Excel、JSON等多种格式。

2

数据清洗

处理缺失值、异常值、重复记录；统一门店编码、品类名称等维度字段；转换时间格式。

3

分析建模

构建日/周/月销售趋势、同环比分析、品类贡献度、时段销售分布等核心指标。

4

可视化呈现

生成动态图表、热力图、仪表盘；支持导出为图片、PDF或交互式HTML。

5

报告生成

自动整合分析结果，生成Word/PPT格式的运营报告，包含数据解读和行动建议。

实战案例：区域业绩分析

完整复现一个真实分析项目的全过程

📋 项目背景

客户：某中式快餐连锁品牌，全国200+门店
需求：分析Q3华东区业绩下滑原因，识别问题门店，制定改进方案
交付周期：3天
数据规模：60万条订单记录，50家门店信息

🔍 分析过程

Step 1：数据探索

# 用户与WorkBuddy对话示例 用户："我有华东区50店Q3销售数据，想分析一下业绩下滑原因。 数据文件：华东区Q3销售数据.xlsx，包含订单时间、门店、金额、品类" WorkBuddy："我来帮你分析。首先让我查看数据结构和基本情况..." → 自动读取Excel文件 → 生成数据质量报告：缺失值、异常值、时间范围 → 输出基础统计：总订单数、总销售额、门店分布

Step 2：趋势分析

# WorkBuddy自动执行的分析 1. 日销售趋势图：识别下滑拐点（7月15日后明显下滑） 2. 同比分析：vs去年同期，整体下滑12% 3. 环比分析：vs Q2，下滑8% 4. 城市对比：杭州(-18%) > 南京(-15%) > 苏州(-8%) > 上海(-5%) 关键发现：7月中旬开始持续下滑，杭州南京下滑最严重

Step 3：根因分析

# 多维度拆解 品类维度： - 套餐类下滑20%（主力产品） - 单品类下滑5% - 饮品类增长8% 时段维度： - 午餐(-15%) > 晚餐(-8%) > 早餐(-3%) 客单维度： - 客流下降10%，客单价下降2% 门店维度： - 问题门店TOP5：杭州西湖店(-35%)、南京新街口(-28%)... 初步判断：主力产品吸引力下降 + 午餐时段流失严重

Step 4：外部因素关联

# 竞品与舆情分析 WorkBuddy自动抓取分析： 1. 竞品活动：7月竞争对手X推出"午市套餐5折"活动 2. 舆情分析：近期差评关键词"涨价"、"分量少"出现频率上升 3. 天气因素：7-8月高温天气，但空调投诉门店业绩更差 根因定位： ① 竞品午餐促销冲击 ② 价格敏感度上升 ③ 部分门店体验问题

请分析【数据范围】的【业务指标】，重点关注【分析维度】。 数据范围：XX区域XX家店的XX时间段数据 业务指标：销售额、订单量、客单价、品类销售等 分析维度：时间趋势、区域对比、门店排名、同比环比等 输出要求： 1. Excel数据透视表 2. 关键发现总结（3-5点） 3. 可视化图表

模板2：报告生成类

基于【数据来源】，生成【报告类型】，面向【受众】。 数据来源：Excel/数据库/已有分析结果 报告类型：日报/周报/月报/专项分析报告 受众：高管/区域经理/加盟商/督导 报告结构： 1. 执行摘要（关键结论前置） 2. 详细分析（数据+图表+解读） 3. 问题识别（风险点标注） 4. 改进建议（可执行方案） 格式要求：PPT/Word/PDF

模板3：自动化脚本类

帮我写一个【功能描述】的Python脚本。 功能描述：自动抓取/数据处理/定时任务等 输入：数据来源、格式、位置 输出：目标格式、存储位置 执行频率：每日/每周/实时 通知方式：邮件/钉钉/企微 附加要求：异常处理、日志记录、容错机制
⚠️ 常见误区
  • 一次性提太多要求：复杂任务分步骤沟通，先做框架再填细节
  • 不提供数据文件：明确告知数据位置和格式，或主动上传文件
  • 期望读心术：行业术语、业务规则要主动解释
  • 不问就问：不确定时先问"你有什么建议"，而不是盲目执行
